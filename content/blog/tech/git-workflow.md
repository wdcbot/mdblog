Git 是目前最流行的分布式版本控制系统。掌握正确的工作流能大大提升团队协作效率。

## Git Flow 工作流

```
main ─────●─────────────●─────────────●
           \           /             /
develop ────●─────●───●─────●───────●
             \   /         \       /
feature ──────●─●           ●─────●
```

## 常用分支

| 分支 | 用途 |
|------|------|
| main | 生产环境代码 |
| develop | 开发主分支 |
| feature/* | 新功能开发 |
| hotfix/* | 紧急修复 |

## 提交规范

推荐使用 Conventional Commits：

```
feat: 添加用户登录功能
fix: 修复首页加载慢的问题
docs: 更新 README
refactor: 重构用户模块
```

好的提交信息让代码历史清晰可追溯。
